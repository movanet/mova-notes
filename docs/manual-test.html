🎯 Quick 3-Step Test (5 minutes)

Step 1: Verify Plugin Loaded (30 seconds)

In Obsidian:

  1. Press Ctrl+Shift+I (opens Developer Tools)
  2. Click Console tab
  3. Type: app.plugins.plugins['auto-deploy']
  4. Press Enter

Expected Result:

{
  githubSettings: {githubOwner: "movanet", ...},
  settings: Settings,
  api: MarkdownRendererAPI,
  ...
}

If you see undefined: Plugin failed to load

  • Look for red errors in console above
  • Take a screenshot and report the error
  • Plugin may need code fixes

Step 2: Check Commands Available (30 seconds)

In Obsidian:

  1. Press Ctrl+P (opens Command Palette)
  2. Type: export
  3. Look for these 3 commands:

Expected commands:

✓ Export and publish current note to GitHub Pages
✓ Export and publish all notes to GitHub Pages
✓ Export current file to HTML (no publish)

If commands don't appear: Plugin commands not registered

  • Check console for errors during plugin load
  • Plugin may have initialization error

Step 3: Test Export (2 minutes)

In Obsidian:

  1. Open note: "Test Plugin.md" (created earlier)
  2. Press Ctrl+P
  3. Run command: "Export current file to HTML (no publish)"
  4. Watch for notifications

Expected Notifications:

📝 Exporting: Test Plugin...
✅ Export complete!

Check Result:

# In terminal, run:
ls -lh D:\Obsidian\obsidianpublish\docs\test-plugin.html

If file exists: ✅ EXPORT WORKS!
If file doesn't exist: Export failed

  • Check console for errors
  • Note the error message
  • Report for debugging

🐛 Troubleshooting

Issue: Plugin Shows undefined

Likely Causes:

  1. Plugin failed to load due to code error
  2. Missing Obsidian API dependencies
  3. TypeScript compilation issue

Debug Steps:

  1. Check console for red errors
  2. Look for "Loading GitHub Pages Publisher" message
  3. If missing, plugin never initialized
  4. Take screenshot of console errors
  5. Report errors for code fix

Issue: Commands Don't Appear

Likely Causes:

  1. Plugin loaded but commands not registered
  2. Error during addCommand() calls
  3. Plugin ID mismatch

Debug Steps:

  1. Check if plugin object exists (Step 1)
  2. If exists, check console for command registration errors
  3. Try: app.commands.listCommands() to see all commands
  4. Search output for "auto-deploy" or "publish"

Issue: Export Runs But No File Created

Likely Causes:

  1. Export path incorrect (should be: D:\Obsidian\obsidianpublish\docs)
  2. File permissions issue
  3. Export code error during HTML generation

Debug Steps:

  1. Check console for errors during export
  2. Verify docs/ folder exists and is writable
  3. Try manual check: ls D:\Obsidian\obsidianpublish\docs\
  4. Look for partial files or error logs

Issue: Publish Fails (Export Works)

Likely Causes:

  1. GitHub token invalid/expired
  2. Network issue
  3. GitHub API error
  4. Repository permissions

Debug Steps:

  1. Check GitHub token in settings
  2. Verify: https://github.com/settings/tokens
  3. Test GitHub access: curl https://api.github.com
  4. Check console for API error messages
  5. Look for HTTP 401/403/404 errors
